树上背包复杂度是 $O(n^2)$ -> $O(nm)$

可以这样想, 对一个点, 合并子树本质是 $m_i$ 的累乘, 或者说是两两合并, 可以拆一下就是一个子树的点和其他子树点的所有进行合并, 这样的合并只在lca进行, 如果点初始 x 的dp权重(就那个状态),
那么最后就 $\sum x_i x_j$, 如果都是 1, 就是 $n^2$

进一步分析, 发现其实是 O(nm)
还是上面那个两两权重乘, 如果对于一个点始终在小于等于 m 子树上 (这是上界,不然这个点有些不会贡献), 那么只用看其他点. 如果某次合并后, 这个点所在子树大于 k, 那么把这个点直接抹除贡献. 于是在此之前, 最多合并了 k 次, 在此之后贡献贡献为 0.

肯定不能抹除所有点的贡献, 还是考虑当子树大于k后里面的点, 认为等效 k 个点, 而这样的子树最多 n / k 个, 此后合并的话, 相当于 这内部两两合并, 就是 n / k * k * k. 假设有散的子树, 大小不是 k, 则可以用上面说的方法进行合并, 合并后这些肯定会变成大于等于 k 的子树, 或者是直接被一个大于等于 k 的吞了. 总会到最后 n / k 个大于等于 k 个子树的状态, 在这之前总贡献为 n * k, 在这之后总贡献还是 n * k
